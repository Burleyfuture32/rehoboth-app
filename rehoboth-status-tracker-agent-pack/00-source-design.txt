## Status colors I recommend

Your original idea is good, but this is cleaner:

- **✅ Green = Complete**
- **🟡 Yellow = Pending / Waiting on someone**
- **🔵 Blue = In Progress**
- **❌ Red = Overdue / Problem**
- **⚪ Gray = Not Started**

That works better than only check, dash, and X, because “pending” and “actively being worked” are not the same thing.

## How it should update

Do not make it purely manual if you can avoid it.

Have the status update from these triggers:

- task completed
- document uploaded
- deadline passed
- milestone marked done
- condition cleared
- note/comment added
- email/request sent
- closing date scheduled

### Example logic
- If docs received and signed → Borrower = **Complete**
- If title ordered but not returned → Title = **In Progress**
- If due date passed and still incomplete → **Overdue**
- If waiting on outside party → **Pending**
- If nothing started yet → **Not Started**

## Recommended fields behind the scenes

For each status item, store:

- `section_name`
- `person_or_company_name`
- `status`
- `notes`
- `assigned_to`
- `due_date`
- `last_updated`
- `updated_by`
- `blocker_reason`
- `percent_complete`
